1.4 NULL в NOT NULL поле — при попытке вставить NULL в поле с ограничением NOT NULL база данных выдаёт ошибку. Это предотвращает появление записей с незаполненными обязательными полями.
1.5 Дублирование первичного ключа — при попытке вставить запись с уже существующим значением PRIMARY KEY база данных выдаёт ошибку. Это обеспечивает уникальность идентификаторов записей.
1.6 Нарушение внешнего ключа — при попытке вставить значение, отсутствующее в родительской таблице, база данных выдаёт ошибку. Это обеспечивает целостность ссылочных данных
